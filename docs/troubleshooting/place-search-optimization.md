# 🛠️ 트러블슈팅 #1 – 대용량 조회 성능 문제

## ⚡ 문제

- 장소 데이터 약 **200만 건**을 적재한 뒤 목록 API 요청 시, 응답이 **최대 20초 이상 지연**
- 키워드 검색, 주소 조인, 인증글 수 집계, 정렬(인기/최신/거리), 대표 이미지까지 **한 쿼리에서 모두 처리**
- 프론트엔드 UX에 심각한 문제 → 발표 직전까지 반드시 해결 필요

## 🔍 원인 분석 (단계별 흐름)

1. **인증글 수 N+1 서브쿼리** → 장소마다 count 서브쿼리 반복
2. **단일 거대 쿼리** → 다중 JOIN + 거리 계산 + 정렬 + 집계 → 실행 계획 비효율
3. **인덱스 부재** → 지역 코드 조건에도 풀스캔 발생
4. **불필요한 JOIN** → 특정 정렬에서 필요 없는 필드도 항상 GROUP BY
5. **거리 계산 직접 수행** → 후보군 축소 전 `ST_Distance_Sphere` 반복 계산
6. **텍스트 검색 RDB 처리** → LIKE 검색/조인으로 DB 부하 심각

## 🔧 시도 & 개선 과정

### 시도 A – 인증글 count 서브쿼리 제거 → 집계 후 조인

- **변경**: `mission_proof`를 `place_id`로 GROUP BY 집계, place와 LEFT JOIN
- **효과**: **20초 → 9~10초**

### 시도 B – 2-phase fetch (ID 선조회 → 상세 조회)

- **변경**:
    - 1단계: 필터·정렬 포함해 **place.id만 조회**
    - 2단계: 해당 ID에 대해서만 JOIN 수행, 필요한 필드만 가져오기
- **효과**: **5초대**

### 시도 C – 인덱스 보강 (지역 코드)

- **변경**: `region_code`, `deleted_at`, 정렬 컬럼 기반 복합 인덱스 추가
- **효과**: 후보군 축소 성능 ↑, 2-phase fetch와 결합 시 체감 속도 개선

### 시도 D – 불필요한 JOIN 제거 (정렬별 분기 처리)

- **문제**: 최신순에서는 인증글 count 불필요하지만 항상 JOIN/GROUP BY 수행
- **변경**: 정렬 조건별로 필요한 JOIN만 수행하도록 분기 처리
- **효과 (실측)**:
    - 최신순: **32ms**
    - 인기순: **2.31s**
    - 인기순 + 검색: **4.93ms**
    - 거리순: **2.54ms**

### 시도 E – Bounding Box 적용 (거리 계산 최적화)

- **문제**: 거리순 정렬 시 `ST_Distance_Sphere` 계산이 모든 후보군에 적용 → 비용 큼
- **변경**: 먼저 Bounding Box로 대략적인 후보군 필터링 후, 소수 후보에만 정밀 거리 계산
- **효과**: 거리 계산 쿼리 성능 개선, 전체 응답 속도 추가 단축

### 시도 F – Elasticsearch 도입 (텍스트 검색 분리)

- **문제**: LIKE 검색/조인 기반 텍스트 검색 → RDB 부하, 검색 지연
- **변경**: Elasticsearch 역색인 기반 검색으로 이전, DB는 집계·정렬 위주로 사용
- **효과**:
    - 텍스트 검색 응답 시간 수백 ms → 수십 ms
    - 복합 쿼리(검색 + 인기순/거리순)도 안정적으로 ms 단위 응답

---

## ✅ 결과

- 성능 개선 단계 요약:
    - **20s → 9~10s → 5s대 → ms 단위**
    - 최신순 32ms, 거리순 2.54ms, 인기순+검색 4.93ms 달성
- 구조 개선으로 프론트 체감 성능 대폭 향상, 발표/데모 안정적 진행 가능
- 단순 기술 도입보다 **구조 리팩터링과 단계별 최적화**가 더 큰 효과

---

## 💡 회고

- 성능 저하의 원인은 단순한 기술 부족이 아니라 **쿼리 구조의 비효율**이었음을 확인
- 새로운 기술을 도입하기보다 **구조를 단순화하고 단계적으로 최적화**하는 방식이 효과적
- 성능 개선은 단발적인 해결책이 아니라 **측정–가설–실험–검증의 반복 과정**임을 경험
- 최적화 과정에서 발생하는 **트레이드오프를 고려하고, 끝까지 문제를 해결하는 책임감**의 중요성을 배움
